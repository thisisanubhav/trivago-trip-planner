## вңҲпёҸ What is this?

Most travel apps make you search for hotels **one city at a time**. Trip Intelligence Planner flips that model entirely.

Plan **Paris вҶ’ Delhi вҶ’ New York** as a single cohesive journey. Dates cascade automatically, hotel options are grouped per destination, and when you're done вҖ” a polished, AI-generated trip brief ties the whole itinerary together.

This project was built to answer one question: *what does hotel discovery feel like when the entire route is the unit of search, not a single destination?*

> рҹҺҜ **Demo-ready out of the box** вҖ” no paid API credits required to explore the full planning experience.

---

## рҹҢҹ Why This Project Stands Out

| Feature | What it means for the user |
|---|---|
| рҹ—әпёҸ **Multi-city first** | The UI is built around a *sequence* of destinations вҖ” not one-off isolated hotel queries |
| рҹҺЁ **Editorial interface** | Feels closer to a premium travel tool than a raw API demo |
| рҹ”„ **Progressive workflow** | Planning вҶ’ Comparison вҶ’ Itinerary вҖ” three clear, distraction-free steps |
| рҹ”’ **Backend-safe architecture** | Anthropic credentials live on the server, never exposed in the browser bundle |
| рҹӣЎпёҸ **Graceful fallback** | No live API? No problem. The app drops into demo data instead of crashing |

---

## рҹ—әпёҸ The Planning Experience

### `Step 1` вҖ” Build Your Route
Add destinations, set the nights you want in each city, and watch check-in/check-out dates **cascade automatically** from stop to stop. No manual date math.

### `Step 2` вҖ” Compare Stays City by City
Hotels are grouped **per destination** вҖ” so you're shortlisting one great option per city, not drowning in an undifferentiated wall of results.

### `Step 3` вҖ” Review Your Trip Budget
The planner calculates total accommodation cost, average nightly rate, and a **city-by-city cost breakdown** before you commit to anything.

### `Step 4` вҖ” Generate a Trip Brief
Selected stays transform into a compact itinerary: timeline, property summaries, and an **AI-written travel brief** (powered by Anthropic when live access is available).

## рҹҡҖ Getting Started

### Prerequisites

- **Node.js** 18+
- **npm**

### 1 В· Clone & Install

```bash
git clone https://github.com/thisisanubhav/trivago-trip-planner.git
cd trivago-trip-planner
npm install
```

### 2 В· Configure Environment

```bash
cp .env.example .env
```

**For demo mode (recommended):**
```env
VITE_USE_MOCK_DATA=true
```

**For live Anthropic-backed requests:**
```env
ANTHROPIC_API_KEY=your_real_key_here
VITE_USE_MOCK_DATA=false
```

### 3 В· Run Locally

```bash
npm run dev
```

| Service | URL |
|---|---|
| вҡЎ Vite Frontend | `http://localhost:3000` |
| рҹ–ҘпёҸ API Backend | `http://localhost:3001` |

## рҹ”җ Why the Backend Matters

Earlier iterations routed model requests through `VITE_ANTHROPIC_API_KEY` вҖ” meaning credentials lived in the browser bundle, visible to anyone who opened DevTools.

The current architecture moves all Anthropic calls to a local Node server. **Your API key never ships to the client.** This makes the project significantly closer to production shape while remaining simple enough to run in one command.
